안녕하세요, patientInfo 팀에 오신 걸 환영합니다 👋

이 매뉴얼은 Java도 Spring도 처음인 팀원이 혼자 힘으로 이 백엔드를 켜고, 코드를 읽고, 작은 수정을 안전하게 반영할 수 있도록 만든 단계별 안내서입니다.

읽는 순서 왼쪽 사이드바의 챕터를 위에서 아래로 읽어주세요. 각 챕터 끝에는 "직접 해보기" 박스가 있고, 그걸 따라하면 자연스럽게 다음 챕터로 넘어갑니다.

이 프로젝트가 하는 일

patientInfo는 환자의 근력 측정 데이터를 병원 임상 환경에서 수집하고 시각화하는 웹 시스템입니다.

  • 측정 디바이스(하드웨어) 와 함께 동작하는 Flutter 앱이 측정값을 백엔드로 전송합니다.
  • 백엔드(이 저장소) 가 데이터를 받아 MariaDB(혹은 PostgreSQL)에 저장합니다.
  • 웹 대시보드(React 프런트엔드, 같은 저장소 frontend/) 가 의료진에게 그래프로 보여줍니다.

여러분이 만지게 될 부분

영역 위치 언제 만지나요
백엔드 (Spring Boot) src/main/java/com/project/urp/ API 추가·수정, 새 컬럼 저장, 비즈니스 규칙 변경
설정 src/main/resources/application.properties DB 접속, 로깅 끄고 켜기, CORS
프런트엔드 (React) frontend/src/ 화면 추가·수정 (이 매뉴얼은 다루지 않음)
배포 Dockerfile, fly.toml Fly.io에 데모 올릴 때

이 매뉴얼이 다루지 않는 것

  • 프런트엔드 React 코드 수정 — 별도 매뉴얼 필요. 지금 단계에서는 백엔드만 다룹니다.
  • 측정 디바이스 펌웨어 — 다른 저장소에 있고 이 팀에서 다루지 않습니다.
  • 의료 데이터 보안 정책 — 보안 감사 결과는 별도 파일 SECURITY_AUDIT.md에 정리되어 있습니다 (워크스페이스 루트).

도움이 필요할 때

  • 먼저 7장 문제 해결을 검색해보세요. 자주 발생하는 에러는 거의 다 있습니다.
  • 모르는 단어는 8장 용어집에서 한글 풀이를 찾을 수 있습니다.
  • 그래도 막히면 팀 채널에 (1) 무엇을 했는지, (2) 무슨 에러가 났는지, (3) 에러 메시지 전체를 함께 적어 질문하세요.
