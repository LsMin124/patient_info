8장. 용어집

매뉴얼이나 코드에서 만난 단어를 한국어로 풀이합니다. 가나다·알파벳 순.

Java / Spring

JDK (Java Development Kit)
Java 코드를 컴파일·실행하기 위한 도구 모음. 이 프로젝트는 17 버전 고정.
JVM (Java Virtual Machine)
Java 바이트코드를 실제로 돌리는 가상 머신. java -jar 명령이 띄우는 것.
Spring Boot
Spring 프레임워크를 "그냥 main 메서드 하나로 실행되게" 묶어 놓은 스타터. 내장 Tomcat·자동 설정·의존성 관리가 핵심.
의존성 주입 (Dependency Injection, DI)
필요한 객체를 직접 new 하지 않고, 컨테이너(Spring)가 생성자 파라미터로 넣어주는 패턴. 결과적으로 클래스가 더 갈아끼우기 쉽고 테스트하기 좋음.
애너테이션 (Annotation)
@RestController, @Service 처럼 @ 로 시작하는 메타 표시. "이 클래스/메서드의 의도가 이것이다" 를 Spring·JPA에 알림.
Lombok
@Getter, @RequiredArgsConstructor 같은 애너테이션으로 getter/setter/생성자를 컴파일 타임에 자동 생성하는 라이브러리. 보일러플레이트가 줄어듦.

데이터 / JPA

JPA (Java Persistence API)
Java 객체와 RDB 테이블을 매핑하는 표준. 구현체는 보통 Hibernate.
Entity
@Entity 가 붙은 클래스. 한 인스턴스가 DB 한 행과 대응. 이 프로젝트의 Patient/Measurement/DataPoint.
Repository
Entity에 대한 CRUD를 위한 인터페이스. Spring Data JPA가 메서드 이름을 보고 구현을 자동 생성.
트랜잭션 (Transaction)
"한 묶음의 DB 작업을 전부 성공 아니면 전부 취소" 단위. @Transactional 메서드가 예외를 던지면 그 안에서 한 모든 DB 변경이 롤백.
LAZY / EAGER fetch
관계 필드(@ManyToOne 등)를 "필요할 때만(LAZY)" 가져올지 "엔티티 로드와 함께(EAGER)" 가져올지의 전략. LAZY가 보통 안전.
N+1 문제
부모 엔티티 N개를 가져오고, 각각의 자식 컬렉션을 LAZY로 N번 더 SELECT 하는 비효율. 페이지가 느려지면 의심해 봐야 함.
ddl-auto
엔티티 정의를 보고 DB 스키마를 자동 변경하는 모드. 값은 none / validate / update / create / create-drop. 이 프로젝트는 운영도 update — SECURITY_AUDIT H-3 참조.
Hikari
Spring Boot의 기본 DB 커넥션 풀. 로그에 자주 등장 (HikariPool-1 - ...).

웹 / HTTP

REST API
HTTP 메서드(GET/POST/PUT/DELETE)와 URL 경로로 자원을 다루는 규약. 이 프로젝트는 /api/v1/... 모두 REST 스타일.
DTO (Data Transfer Object)
요청/응답을 위해 만들어진 단순 데이터 객체. Entity와 분리해서 내부 구조 변경이 외부 컨트랙트를 안 깨도록.
CORS (Cross-Origin Resource Sharing)
브라우저가 "다른 origin(도메인+포트) 의 API를 호출해도 되는가" 를 결정하는 규칙. dev 프로필이 Vite 서버(:5173) 를 허용해 줌.
JSON (JavaScript Object Notation)
{"key":"value"} 형식의 데이터 표기법. API의 요청 본문·응답 본문에 사용.
샌드박스 / 인증 (Authentication)
"누구인지" 를 증명하는 절차. 이 프로젝트는 현재 인증 없음 — 동결 컨트랙트 제한, 자세한 보안 한계는 SECURITY_AUDIT.md.

빌드 / 도구

Gradle
Java용 빌드 도구. build.gradle 에 의존성·태스크가 정의됨.
Gradle Wrapper (./gradlew)
프로젝트에 포함된 Gradle 실행기. 시스템에 Gradle을 설치하지 않아도 지정된 버전으로 빌드 가능.
pnpm
Node.js 패키지 매니저. npm·yarn 보다 디스크·속도가 좋음. 이 프로젝트는 9.15.9 버전 고정.
Vite
빠른 프런트엔드 개발 서버 + 빌드 도구. React SPA의 pnpm dev / pnpm build 가 Vite를 호출.
bootJar
Spring Boot가 만드는 "실행 가능한 단일 jar". 그 안에 모든 의존성과 내장 Tomcat이 들어있어 java -jar 한 줄로 서버가 뜸.

운영 / 배포

Docker
앱과 의존성을 한 이미지로 묶어 어디서나 동일하게 실행되게 하는 컨테이너 기술. 이 프로젝트의 Dockerfile 은 3단계 빌드.
Fly.io
간편 배포용 PaaS. fly deploy 한 번으로 도커 이미지를 빌드·푸시·실행해 줌. 이 프로젝트는 fly.toml 로 설정.
CI/CD
Continuous Integration / Delivery. 코드 push 시 자동으로 빌드·테스트·배포가 돌게 해 놓은 파이프라인. 이 프로젝트는 GitHub Actions (.github/workflows/ci.yml).
환경변수
DB_PASSWORD 처럼 OS/컨테이너 레벨에서 주입되는 값. 비밀번호·API 키 같은 비밀은 코드에 박지 않고 환경변수로 전달.
fly secrets
Fly.io에서 환경변수를 안전하게 저장하는 방법. fly secrets set KEY=VALUE.

의료 / 도메인

PII (Personally Identifiable Information)
개인 식별 정보. 이름·환자ID·생년월일 등. 로그·에러 메시지에 섞이지 않게 항상 주의.
PHI (Protected Health Information)
건강 관련 PII. 측정값·진단 결과 등. 더 엄격하게 다뤄야 함.
측정 세션 (Measurement)
환자 한 명의 한 번의 측정 (시작 → 종료). 안에 여러 개의 DataPoint (시간 + kg 값) 가 들어감.
동결 컨트랙트 (Frozen Contract)
디바이스 펌웨어·Flutter 앱과 미리 합의된 API 모양 (예: measurement_Id 키의 대문자 I). 이 부분은 절대 바꾸지 말 것 — 바꾸면 디바이스가 안 붙음.

이 매뉴얼의 메타 용어

온보딩 (Onboarding)
새 팀원이 프로젝트에 빨리 익숙해지도록 돕는 과정. 이 매뉴얼이 그 도구.
SECURITY_AUDIT.md
워크스페이스 루트의 자동 보안 감사 보고서. 15분마다 갱신. CRITICAL/HIGH/MEDIUM/LOW 등급으로 이슈 정리.
IMPL_SPEC.md / WEB_REBUILD_PLAN.md
이 프로젝트의 결정 기록 (왜 이 구조인가). 신규 팀원은 매뉴얼을 먼저 읽고, 깊은 배경이 필요할 때만 참고.
