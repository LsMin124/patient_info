1장. 개발 환경 준비

노트북에 필요한 도구들을 설치하고, 코드를 받고, 백엔드를 처음 실행해 http://localhost:8080 에서 응답이 오는 것까지 확인합니다. 소요 시간 30분~1시간.

1.1 무엇이 필요한가요

아래 5가지가 노트북에 설치되어 있어야 합니다. 각 항목 옆 "확인 명령" 을 터미널에 쳤을 때 버전 숫자가 나오면 통과입니다.

도구 버전 확인 명령 왜 필요한가
Java (JDK) 17 java -version 백엔드를 실행하는 언어 런타임
Node.js 20 이상 node --version 프런트엔드 빌드용
pnpm 9.15.9 corepack pnpm@9.15.9 --version 프런트엔드 패키지 매니저
Git 2.x git --version 코드를 받기 위해
MariaDB 10.x 이상 mariadb --version 환자 데이터를 저장할 데이터베이스

왜 JDK 17? 이 프로젝트는 build.gradle에서 Java 17을 못 박아두었습니다. 11이나 21로는 빌드가 실패합니다.

도구가 없으면 이렇게 설치하세요

JDK 17

가장 쉬운 방법: SDKMAN! 사용

curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"
sdk install java 17.0.13-tem

또는 Eclipse Temurin 17을 adoptium.net 에서 직접 다운로드.

Node.js + pnpm

nvm으로 Node를 설치하고 corepack을 켭니다.

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
nvm install 20
corepack enable

MariaDB

  • Ubuntu/Debian: sudo apt install mariadb-server
  • macOS: brew install mariadb && brew services start mariadb
  • Windows: 공식 인스톨러 (mariadb.org/download)

1.2 코드 받기

git clone https://github.com/LsMin124/patient_info.git
cd patient_info

1.3 로컬 데이터베이스 만들기

MariaDB에 접속해서 patientinfo 라는 빈 데이터베이스와 전용 계정을 만듭니다. 비밀번호는 본인이 정하세요. 

sudo mariadb -u root

# MariaDB 프롬프트 안에서:
CREATE DATABASE patientinfo CHARACTER SET utf8mb4;
CREATE USER 'patient_dev'@'localhost' IDENTIFIED BY '여기에-본인-비밀번호';
GRANT ALL PRIVILEGES ON patientinfo.* TO 'patient_dev'@'localhost';
FLUSH PRIVILEGES;
EXIT;

주의 위 비밀번호는 본인 노트북 안에서만 쓰는 값입니다. 절대 git에 커밋하지 말고, 팀 채팅·메일에도 보내지 마세요. 노트북 분실 시 즉시 바꿔야 합니다.

1.4 백엔드 설정 파일 만들기

저장소에는 비밀번호를 적은 실제 설정 파일이 들어있지 않습니다. (들어있으면 git에 비밀번호가 공개되니까요.) 대신 템플릿이 있고, 그것을 복사해서 본인 값으로 채워야 합니다. 

cp src/main/resources/application-dev.properties.example \
   src/main/resources/application-dev.properties

그리고 src/main/resources/application-dev.properties 를 에디터로 열어 두 줄을 수정합니다. 

# 수정 전 (템플릿)
spring.datasource.username=${DB_USERNAME:REPLACE_WITH_LOCAL_DB_USER}
spring.datasource.password=${DB_PASSWORD:REPLACE_WITH_LOCAL_DB_PASSWORD}

# 수정 후 (1.3에서 만든 값)
spring.datasource.username=${DB_USERNAME:patient_dev}
spring.datasource.password=${DB_PASSWORD:여기에-본인-비밀번호}

왜 이런 모양? ${DB_USERNAME:patient_dev} 는 "환경변수 DB_USERNAME 이 있으면 그걸 쓰고, 없으면 patient_dev 를 쓴다" 는 의미입니다. 로컬 개발 때는 뒤쪽 기본값이 자동으로 쓰입니다.

1.5 백엔드 처음 실행

저장소 루트에서 다음 명령을 실행하세요. 처음에는 Gradle이 의존성을 내려받느라 2~3분 걸립니다. 

SPRING_PROFILES_ACTIVE=dev ./gradlew bootRun -PskipFrontend=true

이 명령을 풀어쓰면

  • SPRING_PROFILES_ACTIVE=dev — "개발 프로필" 로 켠다 → 방금 만든 application-dev.properties 를 읽는다.
  • ./gradlew bootRun — 프로젝트의 Gradle 래퍼로 Spring Boot 앱을 실행.
  • -PskipFrontend=true — 프런트엔드 빌드는 건너뛰기 (백엔드만 보려면 이게 빠름).

로그가 한참 흐르다가 다음과 같은 줄이 보이면 백엔드가 떠 있는 것입니다. 

Tomcat started on port(s): 8080 (http)
Started UrpApplication in 4.521 seconds

1.6 정말 동작하는지 확인

새 터미널 탭을 열고:

curl http://localhost:8080/api/v1/patients

응답으로 [] (빈 배열) 이 나오면 성공입니다. 아직 환자 데이터를 한 건도 안 넣었으니까요.

🧪 직접 해보기 — 환자 한 명 등록하기

같은 터미널에서:

curl -X POST http://localhost:8080/api/v1/patients \
  -H "Content-Type: application/json" \
  -d '{
    "patientId": "p001",
    "name": "테스트환자",
    "age": 30,
    "sex": "M",
    "height": 170.0,
    "weight": 65.0
  }'

그리고 다시 목록을 조회하면:

curl http://localhost:8080/api/v1/patients

방금 등록한 환자가 보일 거예요. 보이면 1장 끝! 안 보이면 7장 문제 해결 로 가세요.

1.7 백엔드 끄는 법

실행 중인 터미널에서 Ctrl+C 한 번. Gradle이 정리하는 몇 초가 지나면 종료됩니다.
